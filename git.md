**Git** — це потужний інструмент для керування версіями, який використовується для відстеження змін у коді, співпраці в командах і ведення історії проектів. Ось основні аспекти роботи з Git:

---

### 1. **Основні концепції**
- **Репозиторій (Repository)**: Місце, де зберігається код і історія його змін.
- **Гілка (Branch)**: Альтернативна версія коду. Основна гілка зазвичай називається `main` або `master`.
- **Коміт (Commit)**: Фіксація змін у репозиторії з повідомленням для опису.
- **Віддалені репозиторії (Remote Repos)**: Сервери (напр., GitHub, GitLab), де зберігається репозиторій.

---

### 2. **Налаштування Git**
Спочатку потрібно налаштувати своє оточення:
```bash
git config --global user.name "Ваше ім'я"
git config --global user.email "ваш_email@example.com"
```

---

### 3. **Ініціалізація репозиторію**
- **Створення нового репозиторію:**
  ```bash
  git init
  ```

- **Клонування існуючого репозиторію:** (наприклад, з GitHub)
  ```bash
  git clone <url>
  ```

---

### 4. **Основні операції**
#### 4.1. **Відстеження змін**
- **Додавання файлів у staging (підготовчий стан):**
  ```bash
  git add <файл>  # Один файл
  git add .       # Всі файли
  ```

- **Комітування змін:** Фіксація змін із повідомленням.
  ```bash
  git commit -m "Опис змін"
  ```

#### 4.2. **Push та Pull**
- **Завантаження змін у віддалений репозиторій:**
  ```bash
  git push
  ```

- **Отримання змін із віддаленого репозиторію:**
  ```bash
  git pull
  ```

---

### 5. **Гілки**
- **Створення нової гілки:**
  ```bash
  git branch <назва_гілки>
  ```

- **Переключення на гілку:**
  ```bash
  git checkout <назва_гілки>
  ```

- **Створення гілки і переключення:**
  ```bash
  git checkout -b <назва_гілки>
  ```

- **Об’єднання гілок:**
  Перейдіть на основну гілку (`main` або `master`) і виконайте:
  ```bash
  git merge <назва_гілки>
  ```

---

### 6. **Перевірка статусу та історії**
- **Перевірка статусу файлів:**
  ```bash
  git status
  ```

- **Перегляд історії змін:**
  ```bash
  git log
  ```

---

### 7. **Робота з конфліктами**
Під час злиття гілок можуть виникати конфлікти. Вони виникають, коли є суперечливі зміни в одному файлі. У цьому випадку потрібно:
1. Відкрити файл із конфліктом.
2. Виправити конфлікти вручну.
3. Додати файл до `staging` після виправлення:
   ```bash
   git add <файл>
   ```
4. Зробити коміт:
   ```bash
   git commit -m "Виправлено конфлікт"
   ```

---

### 8. **Скидання змін**
- **Відмінити зміни в файлі перед комітом:**
  ```bash
  git checkout -- <файл>
  ```

- **Скинути всі зміни в `staging`:**
  ```bash
  git reset
  ```

---

### 9. **Форк і PR**
- Створіть **fork** через GitHub для копіювання репозиторію.
- Внісши зміни, можна створити **pull request (PR)**, щоб запропонувати зміни автору оригінального репозиторію.

---

### 10. **Git Cheatsheet**
- **Create a new branch:** `git checkout -b <new-branch>`
- **Merge branches:** `git merge <branch-name>`
- **Push changes:** `git push`
- **View status:** `git status`
- **Undo changes:** `git checkout -- <file>`

---
