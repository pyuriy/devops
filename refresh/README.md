# DevOPS refresh 

## Linux
- Освіжити свої знання з адміністрування unix like систем, встановлюючи власні сервіси та розгортаючи Redis, Nginx, RabbitMQ, PostgreSQL, MySql, Wireguard VPN тощо
- Навчитеся писати автоматизацію на bash і використовувати Docker

## CI/CD
- Освоїти основи контролю версій за допомогою Git,
- створити базовий CI/CD пайплайн у GitLab. Ви також
- додати збірку, тести та використання Docker у пайплайні, розширюючи можливості автоматизації розробки та релізу додатків.

## Terraform
Завдання, щоб освоїти основи Terraform для автоматизації інфраструктури як коду (IaC): 
 - встановити Terraform CLI, створить скрипт для розгортання Docker-контейнера з Nginx,
 - налаштувати CI/CD-пайплайн із terraform plan у GitHub Actions,
 - рефакторити код у модулі з дотриманням стилістичних стандартів,
 - інтегруват TFLint для перевірки якості коду
 - розгорнути повноцінний Docker-додаток із фронтендом, бекендом і Nginx як зворотним проксі з самопідписаним SSL-сертифікатом, використовуючи hashicorp/tls провайдер і модульну структуру.

## AWS
- Розгорнути різноманітні ресурси, від базових EC2 та S3 до серверлес-застосунків на Lambda.
- Налаштуєте додаток на EC2, перенесети його на ECS та після зробите його повністю serverless.

## Ansible
Оосвоїти основи Ansible для автоматизації конфігурації: 
 - створити Terraform-скрипт для розгортання EC2-інстансу (або VM через Vagrant),
 - розробити Ansible-playbook для встановлення Nginx і Drone через docker-compose із сертифікатами Let’s Encrypt та автопоновленням,
 - рефакторити playbook у модульні ролі,
 - налаштувти динамічний інвентар EC2,
 - додасти моніторинг через Prometheus, Grafana й Alertmanager на окремих інстансах із роллю monitoring_install для встановлення Nginx як зворотного проксі, інтеграції з CloudWatch Logs для логів, автоматичного додавання нових інстансів до Prometheus та (опціонально) експортера Nginx для збору статистики, забезпечивши функціональність через CI/CD у GitLab.

## Kubernetes
- налаштувати середовище Minikube,
- розгорнути застосунок,
- вивчити комплексні об’єкти Kubernetes та best practices по роботі з Kubernetes,
- навчитися використовувати Helm та Kustomize для повторного використання маніфестів.

## Incident management
- дослідити різні типи надійності (відновлення після катастроф, висока доступність, стійкість до збоїв), визначити їх застосування в реальних сценаріях,
- ознайомитися з культурою постмортем-аналізу, включаючи ключові аспекти управління інцидентами та складання звітів для покращення процесів, продемонструвавши розуміння теми
